# 自动指令功能 (/自动)

## 📋 功能简介

自动指令功能允许用户通过输入 `/自动` 指令，让当前对话的角色连续生成多条对话。这个功能特别适合用于：

- 角色内心独白
- 连续的行动描述
- 动态情节发展
- 丰富对话内容

## 🚀 使用方法

### 基础语法

```
/自动         # 生成3条对话（默认）
/自动 5       # 生成5条对话
/自动 10      # 生成10条对话
```

### 功能特性

- ✅ **智能生成**: 基于角色设定和对话历史生成符合角色特点的对话
- ✅ **登录验证**: 需要用户登录后才能使用，确保安全性
- ✅ **灵活配置**: 支持1-20条对话生成，默认3条
- ✅ **自动保存**: 生成的对话自动保存到聊天历史记录
- ✅ **实时反馈**: 前端实时显示生成进度和结果

## 🔧 技术实现

### 后端 (Python Flask)

- **AutoCommandProcessor**: 核心处理器类
  - `parse_auto_command()`: 解析自动指令
  - `validate_user_session()`: 验证用户登录状态
  - `generate_auto_dialogue()`: 生成自动对话

- **API路由**:
  - `POST /api/auto_command`: 处理自动指令请求
  - `GET /api/auto_command/status`: 获取功能状态
  - `GET /api/auto_command/help`: 获取帮助信息
  - `GET /auto_help`: 显示帮助页面

### 前端 (JavaScript)

- **AutoCommandHandler**: 前端处理器类
  - 自动检测 `/自动` 指令
  - 实时显示生成进度
  - 处理用户界面交互
  - 管理聊天历史更新

### 集成方式

1. **聊天路由集成**: 在 `web/chat_routes.py` 中检测自动指令
2. **蓝图注册**: 在 `web/app_new.py` 中注册自动指令蓝图
3. **前端集成**: JavaScript自动加载到聊天页面

## 🛡️ 安全特性

- **登录验证**: 所有API请求都需要有效的用户会话
- **条数限制**: 最多生成20条对话，防止滥用
- **错误处理**: 完善的异常处理和用户友好的错误提示
- **会话管理**: 使用Flask session确保用户身份验证

```
用户输入: /自动 50

用户输入: /自动abc
系统响应: ❌ 无效的自动指令格式，请使用 /自动 或 /自动 数字

未登录用户: /自动
系统响应: ❌ 用户未登录，请先登录后使用自动指令
```
