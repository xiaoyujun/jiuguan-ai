# 自由事件 - 多角色对话系统

## 功能概述

自由事件是一个让多个角色进行自由对话的系统，支持智能数据书检索、对话总结和记录减负等功能。

## 主要特性

### 1. 多角色选择
- 支持从所有可用角色中选择多个参与者（至少2个）
- 提供搜索功能快速找到需要的角色
- 显示角色头像和名称，直观易用

### 2. 事件配置
- **事件名称**: 为事件命名，方便保存和管理
- **事件背景**: 描述事件的背景和设定，AI会根据此背景生成对话
- **参与角色**: 选择参与对话的角色

### 3. 对话模式

#### 顺序模式（默认）
- 角色按选择顺序依次发言
- 说话概率设置为0时启用此模式

#### 随机概率模式
- 为每个角色设置说话概率（0-10）
- 概率值越高，角色被选中说话的几率越大
- 适合模拟真实的多人对话场景

### 4. 智能数据书集成

利用项目的 `vectorized_temp_data` 系统：
- 自动根据对话内容进行语义搜索
- 智能检索相关角色的数据书信息
- 将相关信息传入AI，使对话更加准确和连贯

### 5. 对话管理

#### 总结到数据书
- 点击"总结到数据书"按钮
- 调用项目现有的总结功能
- 将事件对话内容总结并保存到角色数据书

#### 减负记录
- 当对话记录过长时，可以使用减负功能
- 保留最近20条记录
- 旧记录可选择性总结保存

### 6. 事件保存与加载
- 保存事件到本地JSON文件
- 加载之前保存的事件继续对话
- 事件文件存储在 `聊天记录/事件记录/` 目录

## 使用流程

1. **访问入口**
   - 在聊天输入框输入 `/` 打开斜杠命令菜单
   - 搜索并选择"自由事件"命令
   - 或直接输入 `/free-event` 或 `/自由事件`
   - 也可以直接访问 `/free-event` 路径

2. **配置事件**
   - 输入事件名称和背景描述
   - 搜索并选择参与角色（至少2个）
   - 点击"开始事件"

3. **进行对话**
   - 点击"继续对话"按钮生成下一条对话
   - AI会自动选择下一个说话的角色
   - 对话内容会实时显示在聊天区域

4. **调整概率（可选）**
   - 点击"设置概率"按钮
   - 为每个角色设置说话概率
   - 设置为0则恢复顺序模式

5. **管理对话**
   - 使用"总结到数据书"保存重要内容
   - 使用"减负记录"清理过长的历史
   - 使用"保存事件"保存整个事件状态

## 技术实现

### 后端架构
- **Blueprint**: `event_bp` - 独立的Flask蓝图
- **路由文件**: `web/event/event_routes.py`
- **数据处理**: 使用 `SimpleProcessor` 进行语义搜索

### 前端架构
- **HTML模板**: `web/templates/free_event.html`
- **CSS样式**: `web/static/css/free_event.css`
- **JavaScript**: `web/static/js/free_event.js`

### API接口

| 路由 | 方法 | 说明 |
|------|------|------|
| `/free-event` | GET | 显示自由事件页面 |
| `/api/event/roles` | GET | 获取所有可用角色 |
| `/api/event/create` | POST | 创建新事件 |
| `/api/event/continue` | POST | 继续事件对话 |
| `/api/event/save` | POST | 保存事件到文件 |
| `/api/event/load` | GET | 加载保存的事件 |
| `/api/event/list` | GET | 列出所有保存的事件 |

## 数据格式

### 事件数据结构
```json
{
  "event_name": "事件名称",
  "event_description": "事件背景描述",
  "participants": ["角色1", "角色2", "角色3"],
  "history": [
    {
      "role": "角色1",
      "message": "对话内容",
      "timestamp": null
    }
  ],
  "created_at": null
}
```

## 集成功能

### 与数据书系统集成
- 使用 `vectorized_temp_data` 进行语义搜索
- 自动获取相关角色的数据书信息
- 支持实时更新数据书内容

### 与总结系统集成
- 调用 `/summary` 路由
- 传递事件数据进行总结
- 支持多数据书并发总结

### 与减负系统集成
- 自动保留最近记录
- 可选择性总结旧记录
- 支持手动触发减负

## 页面风格

采用与项目统一的深色主题：
- 渐变紫色主按钮
- 深色卡片背景
- 平滑的动画过渡
- 响应式设计，支持移动端

## 注意事项

1. **认证要求**: 所有事件API端点已添加到认证白名单，无需单独登录
2. **角色数量**: 至少需要选择2个角色才能开始事件
3. **性能考虑**: 对话历史过长时建议使用减负功能
4. **存储位置**: 事件文件保存在 `聊天记录/事件记录/` 目录
