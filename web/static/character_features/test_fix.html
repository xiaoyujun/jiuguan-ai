修复说明

在完全重构数据书功能时，我们移除了复杂的 StorybookManager 类和相关的 updateStorybookTab() 方法。 但是在角色类别切换逻辑中还存在对这个方法的调用，导致了运行时错误。

现在已经修复了这个问题：

  1. 移除了 handleCategoryChange() 中对 updateStorybookTab() 的调用
  2. 在 switchTab() 方法中添加了对AI生成器自动填充功能的调用
  3. 确保AI生成器的 currentCharacter 属性与主系统保持同步
