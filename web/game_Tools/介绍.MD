
# 小游戏系统介绍

## 概述
这是一个集成在AI角色扮演聊天系统中的小游戏页面，旨在为玩家在等待AI语言模型处理时提供娱乐过渡。

## 设计目标
- 提供简易版的扫雷、检定类型小游戏等娱乐内容
- 支持第三方库的GBA或FC游戏集成
- 部分游戏（如D100）可作为检定结果输出到聊天中
- **必须支持移动端设备**

## 技术要求
- 采用API方式与主程序集成，所有集成逻辑集中在单一函数中
- 游戏入口通过斜杠命令触发，格式：`/{game_id}` (如 `/d100`, `/fps`, `/flappybird`)
- 所有小游戏在独立窗口或模态框中运行，支持ESC键关闭
- ✅ **已完成**: 通用游戏分数处理器 (`game_score_processor.py`) 用于处理JavaScript运算的分数结果
- ✅ **已完成**: 自动游戏发现机制，无需手动注册游戏


## 检定输出格式
当游戏用作检定时，输出格式为：
```
(玩家xx进行了一次d100检定)检定结果为:dxx
```

## 当前游戏列表
- **D100骰子** - 经典检定骰子游戏 ✅ 已集成分数处理器
- **FPS定位微调** - 5秒限时精准点击挑战，提升鼠标定位精度和反应速度 ✅ 已集成分数处理器
- **FC红白机游戏** - 基于JSNES的完整FC游戏模拟器，支持ROM加载、状态保存、成就系统 ✅ 已集成分数处理器
- **FlappyBird** - 像素鸟飞行游戏，经典重制版支持移动端 ✅ 已集成分数处理器
- **扫雷** - 扫雷 (待集成)

## 🎯 通用分数处理系统 (已完成)

### 核心功能
- **文件**: `game_score_processor.py`
- **目标**: 将所有游戏分数统一转换为D100格式输出到聊天框
- **原则**: 简化、统一、专注于分数转换

### 支持的游戏转换规则

#### 1. D100骰子 - 直接使用结果
```python
输入: {'result': 85}
输出: d85
消息: (玩家进行了命运之骰)检定结果为:d85
```

#### 2. FPS定位微调 - 分数+准确率转换
```python
输入: {'score': 1200, 'accuracy': 85}
转换算法: (分数/2000)*60 + (准确率/100)*40
输出: d70
消息: (玩家进行了FPS定位微调)检定结果为:d70
```

#### 3. FC红白机游戏 - 时长+成就转换
```python
输入: {'play_time_minutes': 15, 'achievements_count': 2, 'total_achievements': 3}
转换算法: (时长/10)*60 + (成就数/总成就)*40
输出: d87
消息: (玩家进行了FC红白机游戏)检定结果为:d87
```

#### 4. FlappyBird - 分数+成就转换
```python
输入: {'score': 25, 'achievements_count': 3, 'total_achievements': 5}
转换算法: 智能分段转换(0-10分线性,10-50分缓慢,50+渐近) + 成就奖励30%
输出: d70
消息: (玩家进行了像素鸟飞行)检定结果为:d70
```

## 🎮 游戏指令系统

### 统一指令格式
所有游戏现已遵循统一的指令格式：`/{game_id}`

| 指令 | 游戏 | 检定支持 | 说明 |
|------|------|----------|------|
| `/d100` | 命运之骰 | ✅ | 经典D100检定 |
| `/fps` | FPS定位微调 | ✅ | 5秒射击训练 |
| `/flappybird` | 像素鸟飞行 | ✅ | 经典重制版 |
| `/games` | 游戏中心 | - | 打开所有游戏 |

### 自动发现机制
- 系统自动扫描 `game_Tools/` 目录
- 每个游戏目录自动注册为可用指令
- 无需手动配置，新游戏直接可用
- 支持通过 `/help` 查看最新游戏列表
